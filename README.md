## 이 프로젝트를 PM 포트폴리오로 보는 방법

이 프로젝트는 단순한 키워드 분석 도구가 아니라,
콘텐츠 성과를 어떻게 정의하고 판단할 것인가에 대한
PM 관점의 기준 수립 실습을 목적으로 합니다.

- 조회수 중심 KPI의 한계를 문제로 정의하고
- 참여율(ER)을 핵심 판단 지표로 재정의했으며
- 지표 오해 가능성과 QA 관점 체크를 통해
  이후 실험과 의사결정이 흔들리지 않도록 기준을 고정했습니다.

데이터 분석은 결론이 아니라,
**의사결정을 가능하게 하기 위한 수단**으로 사용했습니다.


📈 YouTube 키워드 트렌드 분석기 (yt-trend)
1. 프로젝트 개요

이 프로젝트는 유튜브 콘텐츠 성과를 단순 조회수 중심이 아닌,
사용자 반응 관점에서 구조적으로 해석하기 위해 진행한 데이터 분석 프로젝트입니다.

일반적으로 유튜브 콘텐츠 성과는 조회수로 판단되는 경우가 많지만,
조회수는 노출의 결과일 뿐 실제 사용자의 반응이나 몰입도를 충분히 설명하지 못한다고 느꼈습니다.
이에 따라  참여율(ER = 좋아요 + 댓글 / 조회수) 을 핵심 지표로 설정하고,
콘텐츠의 제목·설명 키워드와 사용자 반응 간의 관계를 분석하는 것을 목표로 했습니다.

2. 문제 정의

조회수가 높은 콘텐츠가 항상 사용자 반응이 좋은 것은 아니다

자극적인 제목은 클릭을 유도할 수 있지만, 실제 참여로 이어지는지는 별도의 문제다

콘텐츠 성과를 판단할 수 있는  일관된 기준(KPI) 이 필요하다

이 프로젝트는 위 문제를 바탕으로
 “어떤 콘텐츠가 사용자에게 실제로 반응을 이끌어내는가?” 라는 질문에서 출발했습니다.

3. 접근 방식
1) 핵심 지표 재정의

조회수(View Count) 대신  참여율(ER) 을 주요 KPI로 설정

단기 노출 성과가 아닌 사용자 반응의 질에 집중

2) 데이터 수집

YouTube Data API를 활용해 키워드·기간·지역별 영상 데이터 수집

영상 제목, 설명, 조회수, 좋아요 수, 댓글 수 등 주요 메타데이터 확보

3) 분석 및 시각화

제목과 설명 텍스트를 토큰화하여 키워드 빈도 분석

워드클라우드를 통해 반복적으로 등장하는 키워드 시각화

조회수 대비 참여율 분포를 통해 콘텐츠 유형별 반응 차이 확인

4. 주요 기능

키워드 / 기간 / 지역 코드 기반 영상 데이터 수집

참여율(ER) 자동 계산

참여율 기준 상위 콘텐츠 정렬

제목·설명 기반 키워드 워드클라우드 시각화

분석 결과 CSV 다운로드 기능

5. 핵심 인사이트
a. 조회수 상위 콘텐츠 ≠ 참여율 상위 콘텐츠

조회수가 높은 영상 중 상당수는 참여율이 낮았으며,
반대로 조회수는 중간 수준이지만 참여율이 높은 콘텐츠도 다수 확인되었습니다.
이는 성과 판단 기준을 재설정할 필요성을 보여줍니다.

b. 자극적인 키워드는 클릭에는 유리하지만, 반응에는 항상 유리하지 않다

강한 표현이나 자극적인 키워드를 사용한 콘텐츠는 조회수 편차가 컸고,
참여율 측면에서는 안정적인 우위를 보이지 않았습니다.

c. 맥락을 설명하는 키워드 조합이 더 안정적인 반응을 만든다

‘하루’, ‘과정’, ‘루틴’과 같이 상황과 흐름을 설명하는 키워드 조합이
참여율이 높은 콘텐츠에서 반복적으로 등장하는 경향을 보였습니다.

d. 참여율이 높은 콘텐츠는 반응의 분산이 낮다

일시적인 바이럴보다, 댓글과 반응이 일정 수준으로 유지되는 콘텐츠가
장기적인 관점에서 더 의미 있는 성과를 만든다고 판단했습니다.

6. PM 관점에서의 해석

이 프로젝트를 통해 단순히 데이터를 시각화하는 데서 그치지 않고,

무엇을 성과로 볼 것인가(KPI 정의)

어떤 지표가 실제 사용자 행동을 설명하는가

단기 성과와 장기 반응을 어떻게 구분할 것인가

와 같은 질문을 구조적으로 다루는 연습을 할 수 있었습니다.

이는 게임이나 모바일 서비스에서
UA 지표, Retention, LTV를 해석하고 A/B 테스트 가설을 세우는 과정과
유사한 문제 구조를 가진다고 생각합니다.

7. 실행 관점에서의 판단 (Next Action)

만약 실제 서비스 환경이었다면,
참여율(ER) 기준 상위 키워드 조합을 중심으로
제목 구조와 썸네일 메시지를 분리한 A/B 테스트를 우선 제안했을 것입니다.
이는 클릭 유도 자체보다,
어떤 요소가 실제 사용자 반응으로 이어지는지를 먼저 검증하기 위한
우선순위 판단이었습니다.
